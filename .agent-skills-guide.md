## 🔒 Güvenlik Check-list (Her Kod Üretiminde)

Aşağıdaki kontroller **her kod yazımından önce** otomatik olarak uygulanır:

### Backend (NestJS)
- [ ] Prisma sorgusunda `tenantId` WHERE clause'da mı?
- [ ] Prisma sorgusunda `deletedAt: null` filtresi var mı?
- [ ] Controller'da `@UseGuards(AuthGuard('jwt'), TenantGuard, RbacGuard)` mevcut mu?
- [ ] Mutasyon işlemlerinde `AuditLog` kaydı oluşturuluyor mu?
- [ ] Birden fazla tabloyu etkileyen işlem `$transaction` içinde mi?
- [ ] DTO'larda `class-validator` decorator'ları var mı?
- [ ] Para tutarları `Decimal(12,2)` tipinde mi?

### Frontend (Next.js)
- [ ] API çağrısında `x-tenant-id` header ekleniyor mu?
- [ ] JWT, HttpOnly cookie'de mi saklanıyor?
- [ ] Para formatı `Intl.NumberFormat('tr-TR', { currency: 'TRY' })` ile mi?
- [ ] Store selector'ları minimum state seçiyor mu (performans)?
- [ ] Server Component / Client Component ayrımı doğru mu?
- [ ] Role-based UI kontrolü yapılıyor mu?

## 🔧 Kullanım Talimatları

### Skill Okuma
Agent bir modül üzerinde çalışırken, ilgili skill'in `SKILL.md` dosyasını otomatik olarak okumalıdır:

```
Örnek: Satış konsolu kodlarken → prisma-expert + financial-algorithm + zustand-state-manager okunur
```

### Öncelik Sırası
1. **Security & RLS Auditor** — her zaman en önce kontrol edilir
2. **Prisma Expert** — backend kod her yazıldığında
3. **Financial Algorithm** — para/stok hesabı olduğunda
4. **shadcn/ui Designer** — UI bileşeni oluşturulurken
5. **Zustand State Manager** — client-side state yönetilirken
